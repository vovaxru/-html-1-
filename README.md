# Вывод HTML-макета в управляемую форму 1С

Пример внешней обработки для платформы «1С:Предприятие 8.3», демонстрирующий правильное извлечение данных из общего HTML-макета конфигурации и их корректное отображение на управляемой форме без потери исходного форматирования (стилей, таблиц и шрифтов).

## 🚀 Реализация проекта

Проект реализован на управляемых формах и состоит из следующих шагов в конфигураторе:

1. **Создание макета**: В ветке `Общие -> Общие макеты` добавлен макет с именем `Макет` и типом данных **HTML документ**.
2. **Создание обработки**: Добавлена новая внешняя обработка с управляемой формой.
3. **Настройка реквизитов формы**: Создан реквизит формы `ПолеСайта` с типом данных *Строка (неограниченная длина)*.
4. **Визуализация**: Реквизит перенесен на форму, для элемента управления установлен вид **Поле HTML документа**.
5. **Логика заполнения**: В модуле формы добавлен обработчик события `ПриСозданииНаСервере` для динамического чтения данных.

## 🛠 Код реализации

Для оптимального извлечения HTML-данных из DOM-модели макета и передачи их в элемент формы используется следующий программный код на стороне сервера:

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
	
	// Извлекаем DOM-структуру документа из общего HTML-макета
	ДокументHTML = ПолучитьОбщийМакет("Макет").ПолучитьДокументHTML();
	
	// Инициализируем потоковую запись HTML
	ЗаписьHTML = Новый ЗаписьHTML;
	ЗаписьHTML.УстановитьСтроку();
	
	// Конвертируем DOM-модель в плоский HTML-текст
	ЗаписьDOM = Новый ЗаписьDOM;
	ЗаписьDOM.Записать(ДокументHTML, ЗаписьHTML);
	
	// Передаем готовый текст с форматированием в реквизит формы
	ПолеСайта = ЗаписьHTML.Закрыть();
	
КонецПроцедуры
```

## 📋 Особенности решения
* **Сохранение стилей**: Метод `ЗаписьDOM.Записать()` гарантирует, что все CSS-стили, разметка и шрифты из конструктора макетов 1С перенесутся на форму без искажений.
* **Работа на сервере**: Чтение тяжелых метаданных макета происходит строго на сервере, минимизируя нагрузку на клиентское рабочее место.
