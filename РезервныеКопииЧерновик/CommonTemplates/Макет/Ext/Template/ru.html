Да, это действительно досадная ситуация. Простите за путаницу: у самого объекта ФорматированныйДокумент в 1С действительно нет метода УстановитьТекст(). Из-за этого платформа и ругается. Он умеет принимать чистый текст только через обходной путь. [1, 2]
Поскольку вы хотите работать без HTML, вот гарантированно рабочий и правильный способ передать обычную строку в форматированный документ.

Рабочий код (исправленный)

Замените весь код внутри процедуры на этот:
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
	// 1. Создаем пустой форматированный документ
	ВременныйДокумент = Новый ФорматированныйДокумент;
	
	// 2. Превращаем обычный текст в форматированную строку (объект 1С)
	// Метод Добавить() создаст правильный элемент внутри документа
	ВременныйДокумент.Добавить("Ваш обычный текст для обработки");
	
	// 3. Переносим готовый документ в реквизит формы
	ЗначениеВРеквизитФормы(ВременныйДокумент, "ТекстЗаметки");
КонецПроцедуры

Как сделать перенос строки в этом методе

Если в таком тексте вам нужно будет сделать абзацы, используйте функцию Символы.ПС:
ВременныйДокумент.Добавить("Первая строка" + Символы.ПС + "Вторая строка");

Почему так?

Объект ФорматированныйДокумент внутри устроен сложнее, чем обычная строка. Метод .Добавить() берет ваш чистый текст и сам упаковывает его в нужный внутренний формат 1С без использования сторонних языков вроде HTML. Теперь код выполнится без ошибок. [3]
